# Required report outline and checklist

## Required outline for this phase

```text
I. ĐỀ XUẤT ĐỀ BÀI
1.1. Giới thiệu vấn đề
1.2. Giải pháp
1.3. Phân công các thành viên tham gia

II. PHÂN TÍCH NGƯỜI SỬ DỤNG VÀ PHÂN TÍCH NHIỆM VỤ
2.1. Giới thiệu
2.2. Phân tích người sử dụng
2.3. Phân tích nhiệm vụ
```

## Completeness checklist

| Requirement | PASS condition |
| --- | --- |
| Topic | Uses HRMS — Trường Đại học Thủy Lợi consistently. |
| 1.1 Problem | Describes problem from user perspective and states user goals. |
| 1.2 Solution | Concise solution, not too detailed; mentions web UI, role-based access, iterative prototype. |
| 1.3 Work allocation | Lists 4 members and concrete tasks/deliverables. |
| 2.1 Intro | Explains purpose of user and task analysis. |
| Interview evidence | Mentions at least 3 representative users or clearly marks missing evidence. |
| 2.2 User groups | Describes Admin, TCCB, TCKT, and Cán bộ/Giảng viên/Nhân viên, unless scope differs. |
| User factors | Covers role, computer skill, domain skill, frequency, needs, and UI implications. |
| 2.3 Task count | At least 5 important tasks. |
| Task details | Each task has goal, precondition, subtasks, and characteristics. |
| HCI grounding | Uses concepts like usability, user goals, task analysis, HTA, frequency, time constraints. |

## Suggested final task set

Use these 6 tasks if the user does not provide a custom set:

1. Tìm kiếm, lọc và xem chi tiết hồ sơ nhân sự.
2. Thêm mới hoặc chỉnh sửa hồ sơ nhân sự.
3. Tạo/chỉnh sửa/chấm dứt hợp đồng lao động.
4. Quản lý tài khoản và phân quyền người dùng.
5. Xem và xuất báo cáo/thống kê nhân sự.
6. Xem hồ sơ cá nhân và đăng ký/hủy đăng ký khóa đào tạo.

Optional replacement/addition:

- Mở khóa đào tạo và ghi nhận kết quả đào tạo.
- Bổ nhiệm, điều chuyển hoặc bãi nhiệm nhân sự khỏi đơn vị.

## Red flags

- “Người dùng là tất cả mọi người” without grouping.
- Task analysis only copies use case names.
- No preconditions.
- No frequency/time constraints.
- Report says “đã phỏng vấn” but no participant/evidence table exists.
- Solution section contains too much backend implementation detail.
