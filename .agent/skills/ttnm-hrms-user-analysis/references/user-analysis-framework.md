# User analysis framework from TTNM slides

## Purpose of user analysis

User analysis is the process of collecting information about users for the first design. It answers:

- Who are the real users of the application?
- What role do users have in the organization?
- What skill level do users have?
- How will users use the application?

The output should identify important user factors and group users with similar characteristics.

## Core analysis factors

Use these factors from the course slides:

| Factor | What to analyze |
| --- | --- |
| Role in organization | Job responsibility and tasks performed. |
| Computer skill | Ability to use mouse, keyboard, browser, forms, GUI. |
| Domain/application skill | Knowledge of HR, contract, finance, training, administration. |
| Cultural/social background | Language, local conventions, institutional context. |
| Frequency of use | Frequent vs occasional use. |

## Extended human factors

Also consider:

- task and user needs: mandatory/optional use, task importance, task structure, social interaction, training needs;
- psychological traits: motivation, patience, pressure level, expectations, thinking style;
- natural traits: age, vision, hearing, dexterity, disability, handedness;
- cognitive constraints: memory load, attention, need for clear feedback;
- reading constraints: readable font, sufficient contrast, simple language.

## Skill level categories

### Computer skill

1. **Người bắt đầu**: little UI knowledge; needs tutorial/help.
2. **Người mới**: some computer experience but lacks experience with this UI; likely to make errors.
3. **Người có kinh nghiệm**: familiar with GUI apps; wants fast interaction.
4. **Người thành thạo**: highly experienced; wants shortcuts and powerful controls.

### Domain skill

1. **Mới bắt đầu**: lacks domain knowledge; needs step-by-step guidance and error recovery.
2. **Người mới**: has some domain knowledge but still makes task errors.
3. **Có kinh nghiệm**: knows the task and performs it well.
4. **Thành thạo/chuyên gia**: deep domain knowledge; wants advanced tools.

## Suggested wording for report

> Phân tích người sử dụng được thực hiện nhằm xác định các nhóm người dùng thực sự của hệ thống, vai trò của họ trong tổ chức, mức độ kỹ năng và ngữ cảnh sử dụng. Kết quả phân tích là cơ sở để thiết kế giao diện phù hợp với mục tiêu, khả năng và hạn chế của từng nhóm người dùng.
