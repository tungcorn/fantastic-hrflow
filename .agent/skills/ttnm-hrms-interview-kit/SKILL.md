# TTNM HRMS Interview Kit

## Purpose

Use this Skill to prepare materials for the report requirement: **interview at least three representative users** and, if possible, observe how they solve the problem in their real work environment.

This Skill produces interview scripts, observation checklists, evidence tables, and synthesis notes for the HRMS topic.

## Required reference loading

Read only what is needed:

- `references/interview-protocol.md` for method and data structure.
- `references/sample-questions.md` for role-specific questions.

## Workflow

1. Identify the representative users to interview. Prefer at least 3 of these roles:
   - Phòng TCCB
   - Phòng TCKT
   - Quản trị viên
   - Cán bộ/Giảng viên/Nhân viên
2. Prepare a short consent/introduction script.
3. Produce role-specific interview questions.
4. Produce an observation checklist for real or simulated tasks.
5. Produce a table for recording answers and converting answers into user/task analysis evidence.
6. If the user has no real interview data yet, clearly label output as **mẫu chuẩn bị phỏng vấn**, not as completed interview evidence.

## Guardrails

- Do not invent real names, dates, or interview answers.
- If sample answers are needed, mark them as “ví dụ minh họa” or “giả định cần xác nhận”.
- Questions must focus on user goals, tasks, pain points, skill level, frequency, tools, errors, and UI expectations.
- Avoid yes/no-only questions; include probes such as “vì sao”, “bước nào mất thời gian nhất”, and “lỗi nào thường gặp”.
