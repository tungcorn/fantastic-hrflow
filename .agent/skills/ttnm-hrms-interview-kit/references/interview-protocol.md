# Interview and observation protocol

## Course-grounded methods

The TTNM slides recommend these methods to understand users:

1. Visit the workplace to understand the real environment.
2. Talk with users about problems, difficulties, expectations, and what currently works well.
3. Establish direct contact with users.
4. Observe users doing their work.
5. Record users performing tasks when appropriate and permitted.
6. Learn about the organization where the system will be installed.
7. Try doing the work yourself to expose hidden difficulties.
8. Prepare surveys/questionnaires for larger samples.
9. Set behavior goals that can be tested.

For this assignment, use a lightweight combination:

- short semi-structured interview;
- observation or task walkthrough;
- evidence table mapping answers to user characteristics and tasks.

## Minimum participant plan

At least 3 representative users are required. Recommended set:

| Participant | Role | Why important |
| --- | --- | --- |
| P1 | Phòng TCCB | Main user for HR records, contracts, appointments, training, evaluations. |
| P2 | Phòng TCKT | Needs accurate lookup/export/statistics for finance-related tasks. |
| P3 | Cán bộ/Giảng viên/Nhân viên | End user for self-service profile and training registration. |
| Optional P4 | Quản trị viên | Account, role, organization, and audit-log management. |

## Data to collect

For each participant, collect:

- role and responsibilities;
- computer skill level: beginner, new, experienced, proficient;
- domain skill level in HR/admin work;
- frequency of system use;
- mandatory vs optional use;
- important tasks;
- current tools used: paper, Excel, email, existing software;
- pain points and errors;
- time pressure and deadlines;
- expected UI support: search, filter, confirmation, error message, help, export, keyboard navigation.

## Observation checklist

When observing a task, record:

| Field | What to record |
| --- | --- |
| Task name | e.g. “Tìm hồ sơ nhân sự theo mã cán bộ”. |
| User goal | What state the user wants to reach. |
| Tools used | Excel, paper file, email, HRMS screen, etc. |
| Steps | Numbered sequence. |
| Time | Approximate time or expected time. |
| Errors/confusion | Missed step, wrong filter, unclear label, data mismatch. |
| Workarounds | Manual notes, asking another department, copying between files. |
| UI implication | What the interface should support. |
